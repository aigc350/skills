# validate 命令

校验一致性。

## 命令格式

```
/novel-creator validate [--fix]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--fix` | false | 自动修复可修复的问题 |

## 执行步骤

1. 检查文件完整性
2. 检查章节连续性
3. 检查设定一致性
4. 检查伏笔状态
5. 输出校验报告

## 检查项

| 检查项 | 说明 | 是否可自动修复 |
|--------|------|----------------|
| 文件完整性 | 所有必需文件是否存在 | 否 |
| 章节连续性 | 章节编号是否连续 | 是（重命名） |
| 设定一致性 | canon 与 canon_ext 是否冲突 | 否 |
| 伏笔状态 | 伏笔是否都有状态记录 | 是（标记为待处理） |

## 输出格式

```
========================================
一致性校验报告
========================================

【文件完整性】✓ 通过
- canon/: 4/4 文件存在
- state/: 7/7 文件存在
- system/: 3/3 文件存在

【章节连续性】✓ 通过
- 已完成章节: 1-50，连续无缺失

【设定一致性】⚠ 发现 2 个问题
- 第 23 章: 角色"张三"行为与设定冲突
- 第 41 章: 地点"神秘森林"未在 world.md 中定义

【伏笔状态】✓ 通过
- 已埋设伏笔: 8 个
- 已回收伏笔: 5 个
- 待回收伏笔: 3 个

========================================
总体状态: ⚠ 发现问题，建议处理
========================================
```

## 错误级别

| 级别 | 图标 | 说明 |
|------|------|------|
| 通过 | ✓ | 无问题 |
| 警告 | ⚠ | 不影响运行，建议处理 |
| 错误 | ✗ | 影响运行，必须处理 |
