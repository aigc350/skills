# test 命令

执行 novel-creator Skill 的自动化测试。

---

## 命令格式

```
/novel-creator test [测试用例]
```

**参数**：
- `测试用例`：指定测试用例文件名（不含 .md 后缀）
- 不指定：使用默认测试用例 `testCase001`

**示例**：
```
/novel-creator test              # 使用默认测试用例
/novel-creator test testCase001  # 指定测试用例
/novel-creator test testCase002  # 使用其他测试用例
```

---

## 执行流程

### 1. 加载测试用例

```
1. 读取 tests/{测试用例}.md 文件
2. 解析测试配置
3. 记录测试开始时间
```

### 2. Phase 1：命令功能测试

按以下顺序执行命令测试：

| 序号 | 命令 | 测试内容 |
|------|------|----------|
| 1 | init | 使用测试用例配置创建小说 |
| 2 | list | 列出小说 |
| 3 | plan | 生成章节规划 |
| 4 | run 1 | 生成单章，测试字数控制 |
| 5 | status | 查看状态 |
| 6 | run 3 | 批量生成3章，测试连续写作 |
| 7 | revise 2 | 修订第2章 |
| 8 | validate | 一致性校验 |
| 9 | export | 导出小说 |

**init 命令特殊处理**：
- 不调用 AskUserQuestion
- 直接使用测试用例中的配置
- 小说ID使用测试用例中指定的值

**每个命令测试后**：
- 记录执行状态（成功/失败）
- 记录执行耗时
- 校验预期文件是否存在
- 发现问题立即记录

### 3. Phase 2：一致性校验

对生成的小说内容进行全面检查：

**2.1 文件结构校验**
- 使用 Glob 检查所有必需文件是否存在
- 记录缺失文件

**2.2 角色一致性校验**
- 读取 `canon/roles.md`
- 读取所有章节文件
- 逐章检查角色行为是否符合设定
- 记录不一致问题

**2.3 地点一致性校验**
- 读取 `canon/world.md`
- 检查章节中的地点是否有定义
- 检查地点移动是否合理
- 记录问题

**2.4 时间线校验**
- 读取 `state/memory.md`
- 检查时间连续性
- 检查时间矛盾
- 记录问题

**2.5 剧情逻辑校验**
- 读取 `canon/outline.md` 和 `state/arcs.md`
- 检查剧情推进是否符合大纲
- 检查因果连贯性
- 记录逻辑漏洞

### 4. Phase 3：质量评估

**4.1 剧情吸引力评估**
- 评估开篇吸引力（10分）
- 评估冲突设置（10分）
- 评估节奏把控（10分）
- 评估悬念设置（10分）
- 评估角色魅力（10分）
- 评估世界观新颖度（10分）

**4.2 写作质量评估**
- 检查 AI 痕迹（10分）
- 检查句式多样性（10分）
- 检查描写恰当性（10分）
- 检查对话自然度（10分）

**4.3 字数达标检查**
- 统计每章字数
- 计算与目标的偏差
- 判定是否合格

### 5. 生成测试报告

将测试结果写入 `novels/{novel_id}/novel/system/test_report_{时间戳}.md`

同时更新 `novels/{novel_id}/novel/system/test_log.md` 记录历史测试结果。

## 校验规则

### 文件存在性校验

```
必需文件 = [
  "canon/meta.md",
  "canon/world.md",
  "canon/roles.md",
  "canon/outline.md",
  "state/state.md",
  "state/chapter_plan.md",
  "state/arcs.md",
  "state/memory.md",
  "state/checkpoint.md",
  "system/run_log.md"
]

for file in 必需文件:
  if not exists(file):
    记录错误("缺失文件: " + file)
```

### 角色一致性校验规则

```
for chapter in chapters:
  for character in chapter.登场角色:
    role = roles.md.查找角色

    if not role:
      记录警告("未定义角色: " + character.name)

    if character.行为不符合(role.性格):
      记录问题("角色行为不一致", 章节, 角色)

    if character.超出能力边界(role.能力):
      记录问题("角色能力越界", 章节, 角色)
```

### 字数校验规则

```
target = meta.md.章节字数  # 如 1800-2200
tolerance = 0.1  # 10%

for chapter in chapters:
  actual = 统计字数
  min_target = target.min * (1 - tolerance)
  max_target = target.max * (1 + tolerance)

  if actual < min_target or actual > max_target:
    记录警告("字数超标", 章节, actual, target)
```
