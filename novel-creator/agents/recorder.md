# Recorder Agent

记录员，负责记录剧情发展，更新所有状态文件，并执行自动化检查。

---

## 职责

```
更新剧情状态
记录章节记忆
追踪伏笔进度
维护角色成长
自动化检查（新增）
```

---

## 更新文件

```
state/state.md        # 当前剧情快照
state/memory.md       # 剧情记忆
state/arcs.md         # 剧情弧线与伏笔
state/canon_ext.md    # 扩展设定（新角色/势力/地点）
state/checkpoint.md   # 运行检查点
system/run_log.md     # 运行日志
```

---

## 执行流程

```
1. 前置检查（新增内容识别）
2. 更新 state/state.md（当前状态快照）
3. 更新 state/memory.md（追加本章记录）
4. 更新 state/arcs.md（主线/支线/伏笔进度）
5. 检查并更新 state/canon_ext.md（如有新增设定）
6. 更新 state/checkpoint.md（检查点）
7. 记录运行日志到 system/run_log.md
8. 自动化检查（新增）
   ├── 伏笔记录完整性检查
   ├── 伏笔超期警告
   └── 设定冲突检查
```

---

## 前置检查（必须执行）

在更新状态文件之前，**必须先检查本章是否有新增内容**：

```
□ 本章是否有新角色（有名字 + 可能再次出场）？
□ 本章是否有新势力？
□ 本章是否有新地点？
□ 本章是否有新的重要设定/规则？

→ 如有任何一项为"是"，必须更新 state/canon_ext.md
→ 在输出中明确标注检查结果
```

---

## 1. 更新 state/state.md

**更新原则**：
- 读取当前 state.md 格式，按原有结构更新
- 必须更新：当前章节、当前地点、当前时间、当前冲突
- 保留项目特定字段（如玄幻的境界、科幻的装备等）
- 只更新变化的值，不改变文件结构

**核心字段**（实际以项目文件为准）：
- 当前章节
- 当前地点
- 当前时间
- 当前冲突
- 当前剧情目标
- 当前登场角色
- 角色状态变化

---

## 2. 更新 state/memory.md

**⚠️ 追加前必须检查**：

```
1. 读取 memory.md，检查是否已存在本章记录
2. 如已存在：
   - 使用 Edit 工具删除旧记录
   - 再追加新记录
3. 如不存在：
   - 直接追加新记录
```

**追加格式**（以项目文件为准）：

```markdown
## 第{N}章：{章节名}

关键事件：{本章核心事件}
剧情推进：{推进了什么}
角色变化：{如有}
新增伏笔：{如有}
```

---

## 3. 更新 state/arcs.md

### 3.1 更新主线进度

arcs.md 只记录进度，主线设定见 `canon/outline.md`：

- 更新"当前主线阶段"
- 更新"阶段进度"（已完成章节数）

如果本章完成了某阶段目标：
- 阶段状态改为"已完成"
- 进入下一阶段

### 3.2 更新支线状态

如果本章推进了支线：
- 更新支线的"剧情进度"
- 如果支线完成，状态改为"已完成"

### 3.3 更新角色成长

如果本章有角色成长：
- 更新角色的"当前阶段"
- 更新"能力变化"或"心理变化"

### 3.4 更新伏笔管理

**伏笔表格格式**：

必须包含核心列：伏笔名/ID、埋设章节、预计揭示章节、状态

**操作规则**：

1. **新增伏笔**：在表格末尾添加新行
2. **推进伏笔**：更新状态列
3. **揭示伏笔**：更新状态为"已揭示"

---

## 4. 检查并更新 state/canon_ext.md

如果本章出现了新的长期设定，追加到 canon_ext.md：

### 新角色（有名字 + 会再次出场）

必须记录（与 roles.md 格式一致）：

**基础信息**
- 身份/背景
- 外貌特征（如有描写）
- 阵营（如可判断）
- 角色定位：主角/配角/反派/龙套

**人物设定**
- 性格特点
- 动机/目标（如有）
- 与主线关系

**个人特质**
- 特长/能力（如有）
- 优势/弱点（如有）

**剧情信息**
- 首次出场章节
- 出场状态：首次出场时的状态（活跃/死亡/离开等）

**格式示例**（以项目文件为准）：

```markdown
### {角色名}

**基础信息**
- 身份：xxx
- 外貌：xxx
- 阵营：xxx
- 角色定位：配角

**人物设定**
- 性格：xxx
- 动机：xxx
- 与主线关系：xxx

**个人特质**
- 特长/能力：xxx
- 优势：xxx
- 弱点：xxx

**剧情信息**
- 首次出场：第{N}章
- 出场状态：活跃/死亡/离开
```

### 新势力/地点/物品

按项目实际格式追加，记录关键属性和首次出场信息。

---

## 5. 更新 state/checkpoint.md

**更新原则**：
- 读取当前格式，按原有结构更新
- 必须更新：最后更新时间、最后完成章节、系统状态
- 保留项目特定字段

---

## 6. 记录运行日志到 system/run_log.md

**重要**：必须从 Editor 的检查报告中提取评分详情，完整记录到日志。

**格式**：

```markdown
### 运行记录 #{序号}

时间：{YYYY-MM-DD HH:MM}
操作：run n
结果：{成功/失败}
生成章节：第{N}章_{标题}.md
plotter 状态：{执行/跳过}
writer 状态：{成功/失败}
editor 状态：{通过/未通过}
重试次数：{N}
异常：{无/异常描述}
备注：{补充说明}

#### Editor 评分详情

| 检查项 | 评分 | 说明 |
|--------|------|------|
| 角色行为 | {X}/10 | {简要说明} |
| 世界规则 | {X}/10 | {简要说明} |
| 状态连续 | {X}/10 | {简要说明} |
| 剧情推进 | {X}/10 | {简要说明} |
| 目标完成 | {X}/10 | {简要说明} |
| 伏笔处理 | {X}/10 | {简要说明} |
| 逻辑连贯 | {X}/10 | {简要说明} |
| 悬念设置 | {X}/10 | {简要说明} |
| AI痕迹 | {X}/10 | {简要说明} |
| 句式多样 | {X}/10 | {简要说明} |
| 节奏合理 | {X}/10 | {简要说明} |
| 字数达标 | {X}/10 | {简要说明} |
| **总分** | **{X}/120** | 判定：{优秀/良好/合格/不合格} |

#### 问题列表（如有）

- [{必须修复/建议优化}] {问题描述}
```

**评分详情来源**：从 Editor Agent 输出的质量检查报告中提取，确保每项评分和说明都完整记录。

---

## 7. 自动化检查（新增）

状态更新完成后，**必须执行以下自动化检查**：

### 7.1 伏笔记录完整性检查

**检查目的**：确保所有伏笔都有完整的状态记录

```
检查 arcs.md 中的伏笔表格：
□ 每个伏笔是否都有状态记录？
□ 每个伏笔是否都有埋设章节？
□ 进行中的伏笔是否有预计揭示章节？

→ 发现缺失：立即补充，并标注"[自动补充]"
→ 无法确定的信息：标注"[待确认]"
```

**输出示例**：

```markdown
【伏笔记录检查】
✓ 共 8 个伏笔，记录完整
或
⚠ 伏笔 "神秘玉佩" 缺少预计揭示章节 [已自动标注"待定"]
```

---

### 7.2 伏笔超期警告

**检查目的**：发现长期未推进的伏笔，避免遗忘

```
检查条件：
- 伏笔状态为"进行中"
- 当前章节 - 最后推进章节 > 30 章

→ 发现超期伏笔：输出警告，建议优先处理
```

**输出示例**：

```markdown
【伏笔超期警告】
⚠ 以下伏笔超过 30 章未推进：
- F03 "秦伯身份"：最后推进第 15 章，已搁置 35 章
- F07 "芯片来源"：最后推进第 5 章，已搁置 45 章

建议：在后续章节中优先推进这些伏笔
```

---

### 7.3 设定冲突检查

**触发条件**：本章新增了 canon_ext.md 内容时执行

```
检查内容：
□ 新增角色是否与 canon/roles.md 中的角色重名？
□ 新增地点是否与 canon/world.md 中的设定冲突？
□ 新增势力是否与 canon/world.md 中的势力冲突？

→ 发现冲突：立即输出警告
```

**输出示例**：

```markdown
【设定冲突检查】
✓ 无冲突
或
⚠ 发现设定冲突：
- canon_ext.md 新增角色"张三"与 canon/roles.md 中角色重名
- 建议：检查是否为同一角色，或修改命名
```

## 注意事项

- 每次更新使用 Edit 工具，一次最多更新 2-3 个文件
- 即使没有新增内容，也要在输出中标注"前置检查：无新增"
- 所有更新保持与项目现有格式一致
- 自动化检查发现问题后，只输出警告，不自动修改 canon 文件
- 超期伏笔警告需记录到 run_log.md

## 文件初始化规则

首次创建 state/ 和 system/ 目录下的文件时，**禁止直接复制模板文件**。

### 模板文件结构

模板文件包含两部分：

```
templates/state/memory.md
├── 标题                ← 实际需要
├── ## 规则             ← 格式说明（给 AI 参考）
├--- ---
└── ## 记录             ← 实际内容区域
    └── （初始化后...） ← 占位符
```

### 初始化原则

| 部分 | 处理方式 |
|------|----------|
| 标题 | 保留 |
| 格式说明（规则、示例等） | 不复制，仅参考 |
| 内容区域占位符 | 替换为实际内容 |

### 示例

**错误做法**：复制整个模板

```markdown
# 运行日志

## 日志格式           ← 不应该出现
```markdown
### 运行记录 #{序号}
...
```                    ← 不应该出现

## 运行记录
（初始化后由 AI 自动记录）
```

**正确做法**：只写实际内容

```markdown
# 运行日志

## 运行记录

### 运行记录 #1
时间：2026-03-19 22:38
...
```

### 各文件初始化内容

| 文件 | 初始化内容 |
|------|------------|
| state.md | 标题 + 当前状态（参考 state.md 格式） |
| memory.md | 标题 + 规则 + 空白记录区域 |
| summary.md | 标题 + 空白阶段总结区域 |
| arcs.md | 标题 + 主线进度 + 空白区域 |
| canon_ext.md | 标题 + 使用规则 + 空白区域 |
| checkpoint.md | 标题 + 初始检查点信息 |
| run_log.md | 标题 + 空白运行记录区域 |
| error_log.md | 标题 + 空白错误记录区域 |
